## 概要

BinOp (びんおーぴー) は、2項演算子 (BINary OPerator) を文法の中心に据えた動的型付けプログラミング言語です。
関数型言語とプロトタイプベースのオブジェクト指向言語、両方の特徴を持ちます。

---

## 特徴

### 特徴1: 関数型 × プロトタイプオブジェクト指向

- Io 言語ライクなプロトタイプベースのオブジェクト指向言語です。
- 再帰とリスト構造をベースにした関数型言語の側面も持ちます。
- 変数に型はありませんが、値には型があります。

### 特徴2: 2項演算子を中心に置いた文法

- `+`, `-`, `*`, `/`, `=`, `==` といった一般的な演算子に加え、`;` や `,` も2項演算子として扱います。
- 単項演算子 (`+1`, `-1`, `!1`, `x++`) や配列アクセス (`ary[1]`) は存在しません。
- `for` / `while` 文もなく、繰り返しは再帰で表現します。
- 構文的に2項演算子でないものは、数値・文字列・括弧・変数を含むレシーバのないメッセージのみです。

### 特徴3: シンプルな文法とコンパクトなパーサ

- 文法ルールがシンプルなため、構文解析器の実装コストが小さく抑えられています。
- 2項演算子の追加が容易な設計になっています。

### 特徴4: シンプルな文法の上に豊かな表現力

- マクロやユーザ定義2項演算子などの高度な機能を持ちます。
- 文法要素を追加することなく、オブジェクト指向プログラミングが実現されています。

### 特徴5: 実用性皆無

- 初心者向けでも、高度な問題をスマートに解くためのものでもありません。
- 純粋に言語実装の探求を目的として作られています。

## チュートリアル

### 真偽値・数値・文字列

#### 真偽値

`nil` だけが偽で、それ以外はすべて真です。比較演算子は、偽の場合 `nil`、真の場合非 `nil` を返します。

```
nil; "唯一の偽"
0; 1; ""; "hello"; "これらはすべて真"
0>1; "比較演算子の結果はnil"
```

#### 数値

0 と正の整数のみ即値で表現できます。負数や小数は計算で作ります。

```
0; 1; 2; 10; "有効な数値"
0-1; "負数は計算で作る"
1/4; "小数も計算で作る"
```

#### 文字列

ダブルクォーテーションで囲まれた範囲が文字列です。連結や部分文字列の取得もできます。

```
"これは文字列です"
"ab" + "cd"; "=> abcd"
"ab" + 12;   "=> ab12"
"abcd" / 2;  "=> ab  (前半)"
"abcd" % 2;  "=> cd  (後半)"
```

---

### コメント

BinOp にコメント構文はありません。評価されない位置に文字列を置くことでコメントとして機能させます。

```
"これはコメントとして機能する文字列です"
```

### 文法

#### 変数

`:=` で宣言し、`=` で再代入します。

```
v := 1; "変数の宣言"
v = v + 5; "再代入"
v.print(); "=> 6"
```

#### 2項演算子の優先順位

すべて左結合で、数字が小さいほど結合が強いです。

| 優先度 | 演算子 |
|---|---|
| 0 | `.` |
| 1 | `*` `/` `%` |
| 2 | `+` `-` |
| 3 | `<` `>` `<=` `>=` |
| 4 | `==` `!=` |
| 5 | `&&` `\|\|` |
| 6 | `:=` `=` |
| 7 | ユーザ定義2項演算子 |
| 8 | `;` |
| 9 | `,` |

```
n := 1 + (2 - 3) * 4; "=> n.:=(1.+((2.-(3)).*(4))) と解釈される"
n.print(); "=> -3"
```

#### セミコロンと改行

セミコロンと改行はどちらも順次処理を行う2項演算子 `;` です。
`a;b` は `a.;(b)` と解釈され、`a` → `b` の順に評価され、最後に評価した `b` の値が返却されます。
改行はセミコロンとして読み替えられ、連続・過剰なセミコロンは読み飛ばされます。

```
"first".print();"second".print(); "=> first, second の順に表示"
```

#### 条件分岐

条件分岐は `if` メソッドで行います。レシーバが `nil` であれば偽ブランチ、それ以外であれば真ブランチに分岐します。

```
(n==7).if(
  "seven",
  "not seven"
).print(); "=> not seven"
```

### 関数

`fun` で関数オブジェクトを生成します。最後の引数が関数本体で、最後に評価した値が戻り値になります。

```
addPrint := fun(a, b,
  result := a + b;
  result.print();
  result);
addPrint(3, 6); "=> 9 と表示し、9 を返す"
```

#### 繰り返し処理

ループ構文はなく、再帰で繰り返しを表現します。

```
factorial := fun(n,
  (n>1).if(
    n * factorial(n-1),
    1))
factorial(4).print(); "=> 24"
```

`fun` オブジェクトには組み込みメソッド `doWhile` もあります。

```
v := 0
z := fun(v<5).doWhile(
  v.print();
  v = v+1
); "=> 0,1,2,3,4 と表示"
```

#### クロージャ

関数は外部スコープの変数にアクセスできます。

```
makeCounter := fun(n:=0; fun(n=n+1))
counter := makeCounter()
counter().print(); "=> 1"
counter().print(); "=> 2"
```

### データ構造（オブジェクト・リスト）

#### オブジェクトの作成とクローン

`Object` は組み込みの基底オブジェクトです。`clone` でコピーを作り、スロット（プロパティ）を追加することでクラスのように扱います。
clone されたオブジェクトは clone 元の参照を持ち、自身に定義されていないメッセージを受け取った場合は clone 元を辿って応答します。

```
Dog := fun(name, age,
  dogClass := Object.clone()
  dogClass.name = name
  dogClass.age := age
  dogClass.greet = fun("I'm " + this.name)
  dogClass)

dog1 := Dog("Foo", 10)
dog2 := dog1.clone()
dog2.name = "Bar"
dog1.greet().print(); "=> I'm Foo"
dog2.greet().print(); "=> I'm Bar"
```

#### ポリモーフィズム

変数に型がないため、クラスの継承関係にかかわらず同名メソッドを呼び出せます。

#### 継承

clone チェーンによって基底クラスのメソッドを継承できます。

```
BaseClass := Object.clone()
BaseClass.func := fun(1+2)
SubClass := BaseClass.clone()
instance := SubClass.clone()
instance.func().print(); "=> 3"
```

#### カプセル化

クロージャを使うと、クラス内部の変数を外部から隠蔽できます。

#### リスト構造

組み込みのデータ構造は `Object` のみのため、リスト構造は自分で定義します。

```
Cell := fun(a, b,
  cellClass := Object.clone()
  cellClass.head := a
  cellClass.tail := b
  cellClass)

list := fun(i, n,
  (i<n).if(Cell(i, list(i+1, n)), nil))

sum := fun(lst,
  lst.if(lst.head + sum(lst.tail), 0))

numbers := list(0, 5)
sum(numbers).print(); "=> 10"
```

### メッセージング

> **注**: メッセージの生成・評価 (`message`, `evalNode`, `evalStr`) は **go-compiler-vm では未サポート**です。ts-interpreter および go-interpreter でのみ利用できます。

BinOp のすべてのプログラムはメッセージです。例えば `1+2` は `1.+(2)` と解釈され、`1` をレシーバに `+(2)` というメッセージが送られます。

#### メッセージの生成

`message` 関数でメッセージオブジェクトを生成できます。

| オプション | 意味 |
|---|---|
| `"__"` | レシーバなし、呼び出しなし |
| `"_@"` | レシーバなし、呼び出しあり |
| `"@_"` | レシーバあり、呼び出しなし |
| `"@@"` | レシーバあり、呼び出しあり |

```
message("_@", "method", 1, 2, 3); "=> method(1,2,3)"
message("@@", receiver, "method", 1, 2, 3); "=> receiver.method(1,2,3)"
```

#### メッセージの評価

`evalNode` で生成したメッセージを評価（実行）できます。`evalStr` で文字列から直接プログラムを評価することも可能です。

```
evalNode(message("@@", 5, "+", 7)); "=> 12"
evalStr("5+7"); "=> 12"
```

### ユーザ定義2項演算子

引数を1つ取るメソッドは、スペース区切りの2項演算子として呼び出せます。

```
o := Object.clone()
o.setValue = fun(x, this.value = x)
o setValue 123; "o.setValue(123) と解釈される"
o.value.print(); "=> 123"
```

### マクロ

> **注**: `macro` は **go-compiler-vm では使用できません**（コンパイル時にエラーになります）。ts-interpreter および go-interpreter でのみ利用できます。

`macro` で引数の評価を遅延させる関数を定義できます。引数は評価されずに渡され、`evalNode` で任意のタイミングで評価できます。

```
while := macro(cond, block,
  f := fun(cond, block,
    evalNode(cond).if(
      evalNode(block);
      f(cond, block),
      nil));
  f(cond, block))

x := 0
while(x<10,
  x.print();
  x = x+1) "=> 0〜9 を表示"
```
